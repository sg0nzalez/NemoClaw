Use this page when you operate an inference server other than Ollama.
It helps you select the setup workflow for compatible servers, vLLM, or NVIDIA NIM.
The agent inside the sandbox connects through `inference.local`, not directly to your self-hosted server.
OpenShell intercepts inference traffic and forwards it to the endpoint you configure.

## Prerequisites

<AgentOnly variant="openclaw">
- Install NemoClaw first.
  Refer to the [Quickstart](../get-started/quickstart) if needed.
</AgentOnly>
<AgentOnly variant="hermes">
- Install NemoClaw first.
  Refer to [Quickstart with Hermes](../get-started/quickstart) if needed.
</AgentOnly>

Start a local model server, or use a supported vLLM or NIM setup that the onboard wizard can install or start.

## Choose a Provider

- [Set Up Other Self Hosted Servers](local-compatible-inference-setup/set-up-other-self-hosted-servers) when you already operate an OpenAI-compatible or Anthropic-compatible server, including TensorRT-LLM, llama.cpp, LocalAI, or a raw GGUF model server.
- [Set Up vLLM](local-compatible-inference-setup/set-up-vllm) when you run vLLM yourself or want NemoClaw to manage vLLM on a supported NVIDIA GPU host.
- [Set Up NVIDIA NIM](local-compatible-inference-setup/set-up-nvidia-nim) when you want NemoClaw to manage a local NVIDIA NIM container.

## Verify and Tune

Use [Verify Self Hosted Inference](local-compatible-inference-setup/verify-self-hosted-inference) after onboarding to check the sandbox route and configure inference timeouts.
